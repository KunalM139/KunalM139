## Featured Projects

<details open>
<summary><b>Flame Detection and Fire Alarm System</b></summary>

<br>

Developed an IoT-based flame detection and fire alarm system using ESP32 cameras, Raspberry Pi, OpenCV, smoke sensing, and temperature sensing for real-time emergency monitoring.

| Attribute | Details |
|---|---|
| Stack | IoT, ESP32, Raspberry Pi, OpenCV, smoke sensors, temperature sensors |
| Scale | Real-time edge monitoring system for fire and smoke detection scenarios |
| Performance | Designed for faster detection through local computer vision processing and sensor fusion |
| Security | Multi-layer emergency response architecture with alert-oriented system flow |
| Impact | Reduced alert latency by combining visual flame detection with environmental sensor checks |
| Repository | [View repositories](https://github.com/KunalM139?tab=repositories) |

The project combines hardware, edge processing, and emergency response design to detect fire conditions earlier than single-sensor systems. It uses OpenCV-based flame analysis with smoke and temperature signals to trigger more reliable emergency alerts.

</details>

<details open>
<summary><b>ResQLink - Emergency Mesh Communication System</b></summary>

<br>

Built an Android emergency communication app using Kotlin, Firebase, BLE mesh networking, GPS tracking, encrypted SOS transmission, and delay-tolerant relay mechanisms.

| Attribute | Details |
|---|---|
| Stack | Kotlin, Firebase, BLE, Android Studio, GPS |
| Scale | Offline emergency communication system for disaster and low-connectivity environments |
| Performance | Uses opportunistic relay and store-carry-forward delivery for resilient message transmission |
| Security | Includes encrypted SOS transmission and controlled emergency alert flow |
| Impact | Enables emergency messages even without internet or cellular connectivity |
| Repository | [View repositories](https://github.com/KunalM139?tab=repositories) |

ResQLink is designed for crisis environments where normal communication infrastructure may fail. The system uses BLE mesh communication to relay emergency messages across nearby devices, improving the chance of message delivery during disasters.

</details>
